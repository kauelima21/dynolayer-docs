Introdução

No DynoLayer, é possível persistir dados de duas formas: fazendo um create ou um save.

Abaixo, haverão exemplos de como implementar ambos os casos.

Salvando registros

A primeira forma de persistência é usando o create. Para isso, é necessário antes que hajam propriedades declaradas no atributo fillable. Uma vez que estas tenham sido indicadas, basta acionar o método passando um dicionário que as contenha. O DynoLayer filtrará pelos campos declarados e realizará o cadastro do registro.

A segunda forma é usando o save, que também é usada para atualizar registros. Ou seja, caso o registro não exista com a chave primária indicada, o DynamoDB irá inserir um novo item na tabela.

Atualizando registros

Conforme o exemplo anterior, o método save também é utilizado para atualizar registros em uma tabela. Desse modo, ao utilizar o save em um item que já existe, a operação apenas atualizará os campos passados.
