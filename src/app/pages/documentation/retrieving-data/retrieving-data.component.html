Introdução

Existem algumas formas de realizar consultas com o DynoLayer, onde, dependendo do caso de uso, será aplicado uma operação de query ou scan contra a tabela.

Abaixo, haverão exemplos de como implementar ambos os casos.

Resgatando todos o itens da tabela

Para realizar um "full scan", usa-se o método all, que realizará uma operação de scan paginando todos os registros e retornando uma lista de dicionários.

Realizando busca por chave primária

Para resgatar um registro através de sua chave primária (simples ou composta), usa-se o método find, o qual retorna uma nova instância da model com os dados resgatados.

Buscando com filtros

Para realizar operações de filtro, usa-se o método where, com o qual é possível construir um query builder encadeando métodos.

Ao final, deve-se especificar o método fetch para queries ou p get para scans.
Caso os campos usados nos filtros não façam parte de nenhuma chave, o método chamado será o get.
